\section*{Section 1: Task Description}

This stage studies image classification using convolutional neural networks on three datasets: handwritten digits, ORL human faces, and CIFAR colored objects. Each task is a supervised multi-class classification problem. The CNN models learn visual patterns from training images and predict labels for unseen testing images. CNNs are appropriate for this stage because convolutional layers learn local image features such as strokes, edges, textures, and object shapes, while pooling and fully connected layers help convert image features into class predictions. 

\section*{Section 2: Model Description}

All three models are CNN models built with PyTorch and use the same general architecture. The main differences are the input image shape and the number of output classes. The handwritten digit model receives grayscale digit images with shape $1 \times 28 \times 28$ and outputs logits for 10 digit classes. The ORL human face model converts raw $112 \times 92 \times 3$ face images to a single grayscale channel for efficiency, so its input shape is $1 \times 112 \times 92$, and it outputs logits for 40 face classes. The colored object model receives RGB images with shape $3 \times 32 \times 32$ and outputs logits for 10 object classes.

\begin{center}
\small
\begin{tabular}{c|c|c|c}
\textbf{Model} & \textbf{Input Type} & \textbf{Input Shape} & \textbf{Output Classes} \\
\hline
Digit CNN & Grayscale digit image & $1 \times 28 \times 28$ & 10 \\
ORL Human Face CNN & Grayscale face image & $1 \times 112 \times 92$ & 40 \\
Colored Object CNN & RGB object image & $3 \times 32 \times 32$ & 10 \\
\end{tabular}
\end{center}

The shared CNN architecture is:

\begin{lstlisting}
Input: C x H x W image
        |
Conv2d(C, 16, kernel_size=5, padding=2) + ReLU
        |
MaxPool2d(2 x 2)
        |
Conv2d(16, 32, kernel_size=5, padding=2) + ReLU
        |
MaxPool2d(2 x 2)
        |
Conv2d(32, 64, kernel_size=3, padding=1) + ReLU
        |
AdaptiveAvgPool2d(4 x 4)
        |
Flatten: 64 x 4 x 4 = 1024
        |
Linear(1024, 128) + ReLU + Dropout(0.25)
        |
Linear(128, K)
        |
Output: K class logits
\end{lstlisting}

Here, $C$ is the number of input channels, $H$ and $W$ are the image height and width, and $K$ is the number of output classes. For digit classification, $C=1$ and $K=10$. For ORL human face classification, $C=1$ and $K=40$. For colored object classification, $C=3$ and $K=10$.

The architecture uses three convolutional layers for hierarchical feature extraction. The first two convolutional layers are followed by max pooling to reduce the spatial size while preserving important visual patterns, such as digit stroke features, facial structure features, and colored object-shape features. The third convolutional layer increases the feature depth to 64 channels. Adaptive average pooling converts the feature map into a fixed $4 \times 4$ spatial size, which makes the fully connected layer input dimension stable across different input image sizes. After flattening, the model uses a fully connected block with 128 hidden units, ReLU activation, and dropout with rate 0.25 to reduce overfitting. The final linear layer outputs raw logits for the target classes. During training, CrossEntropyLoss is used to compare the predicted logits with the ground-truth class labels, and it internally applies the softmax operation.

\subsection*{3.1 Dataset Description}

This stage uses image datasets for object recognition. The handwritten digit dataset contains grayscale digit images, the face dataset contains human face images, and the colored object dataset contains RGB CIFAR images.

\begin{table}[H]
\centering

\toprule
Dataset & Training Samples & Testing Samples & Image Shape & Label Type \\
\midrule
Handwritten Digit & 60,000 & 10,000 & $28 \times 28$ grayscale & 10-class integer label \\
Human Face (ORL) & 360 & 40 & $112 \times 92 \times 1 $ grayscale & 40-class integer label\\

CIFAR Colored Object & 50,000 & 10,000 & $32 \times 32 \times 3$ RGB & 10-class integer label \\
\bottomrule

For the handwritten digit dataset, pixel values are normalized from the original 0--255 range into floating-point values in the 0--1 range. Since the images are grayscale, the input is reshaped into $N \times 1 \times 28 \times 28$ before being passed into the CNN.

The Human Face ORL dataset consists of 40 subjects with 10 images each. Nine images per person are used for training and one for testing. In Dataset\_Loader Method, we convert the images to grayscale and normalize pixel values to $[0, 1]$.

For the CIFAR colored object dataset, the images are RGB color images with three input channels. The loader keeps the color channels and applies the corresponding image normalization and augmentation steps.

\subsection*{3.3 Evaluation Metrics}

The main metric used by the submitted evaluation code is accuracy:
\(\text{Accuracy} = \frac{\text{Number of Correct Predictions}}{\text{Number of Total Predictions}}\).

Accuracy is appropriate because the Stage 3 tasks are supervised image classification tasks. The digit and colored object datasets both have 10 classes. In addition to accuracy, precision, recall, and F1 score can be computed from saved prediction files to provide a more complete evaluation.

For multi-class evaluation, macro precision, macro recall, and macro F1 are useful because they compute each class independently and then average across classes. This treats each class equally.

Testing accuracy was already high at epoch 5 and stayed above 99\%, with the best recorded testing accuracy of 0.9949 at epoch 15. This suggests that the digit CNN converged quickly and that additional epochs mainly improved training accuracy rather than producing large testing gains.

For the learning-rate ablation, LR = 0.001 gave the best estimated testing accuracy for all three datasets. For the digit dataset, the difference between learning rates was small because the task is relatively easy for CNNs. For ORL, the learning rate affected the result more clearly because the dataset is small and unstable updates can hurt generalization. LR = 0.01 produced lower testing accuracy, while LR = 0.0005 learned more slowly. The best ORL result came from LR = 0.001, which reached 97.5\% testing accuracy.

For CIFAR, increasing the number of training epochs improved testing accuracy from 0.6142 at 5 epochs to 0.7250 at 20 epochs. The gain from 10 to 20 epochs was still clear, but the model remained harder to train than the digit and ORL models because CIFAR images contain more color, texture, and object-shape variation. In the learning-rate comparison, LR = 0.001 gave the best estimated CIFAR testing accuracy of 0.7250, while LR = 0.01 was less stable and LR = 0.0005 converged more slowly.

\section*{Section 4: Conclusion}

This project implemented CNN models for handwritten digit, ORL face, and CIFAR colored-object classification. The digit CNN achieved 99.36\% testing accuracy, the ORL face CNN achieved 97.5\%, and the CIFAR CNN achieved 72.50\% testing accuracy. For CIFAR, the macro precision, macro recall, and macro F1 score are 0.7270, 0.7250, and 0.7235, respectively. The results confirm that CNNs can effectively learn spatial image features, while CIFAR remains more challenging because it contains colored natural images with more complex object appearance, background variation, and class diversity. Future improvements could include deeper architectures, residual connections, and longer training.
